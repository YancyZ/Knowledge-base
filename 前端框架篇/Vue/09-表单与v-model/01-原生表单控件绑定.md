# 原生表单控件绑定

原生 **v-model** 随控件类型切换 value/checked 与 input/change，checkbox 分布尔/数组，radio 靠 value，select 多选要 `multiple`。搞清这张表，自定义组件协议才好对齐。

---

## v-model 本质

```vue
<input v-model="text" />
<!-- 等价于 -->
<input :value="text" @input="text = $event.target.value" />
```

Vue 3 对不同类型元素使用不同 **DOM 属性与事件**：

| 元素 | 绑定属性 | 事件 |
|------|----------|------|
| text input / textarea | value | input |
| checkbox（单） | checked | change |
| checkbox（多选组） | checked + value | change |
| radio | checked | change |
| select | value | change |

---

## 文本类 input

```vue
<script setup>
import { ref } from 'vue'

const name = ref('')
const bio = ref('')
</script>

<template>
  <input v-model="name" placeholder="姓名" />
  <textarea v-model="bio" rows="4" />
</template>
```

`textarea` 不用子节点插值，始终用 `v-model`。

## checkbox

**单个布尔：**

```vue
<script setup>
const agreed = ref(false)
</script>
<template>
  <input type="checkbox" v-model="agreed" /> 同意协议
</template>
```

**多个值绑定到数组：**

```vue
<script setup>
const picked = ref([])
</script>
<template>
  <input type="checkbox" value="a" v-model="picked" />
  <input type="checkbox" value="b" v-model="picked" />
  <!-- picked: ['a','b'] -->
</template>
```

---

## radio

```vue
<script setup>
const size = ref('m')
</script>
<template>
  <input type="radio" value="s" v-model="size" />
  <input type="radio" value="m" v-model="size" />
  <input type="radio" value="l" v-model="size" />
</template>
```

`value` 属性决定选中时写入 `size` 的值。

---

## select

**单选：**

```vue
<script setup>
const city = ref('')
const cities = ['北京', '上海', '深圳']
</script>
<template>
  <select v-model="city">
    <option disabled value="">请选择</option>
    <option v-for="c in cities" :key="c" :value="c">{{ c }}</option>
  </select>
</template>
```

**多选（需 multiple 属性）：**

```vue
<select v-model="selected" multiple>
  <option v-for="id in ids" :key="id" :value="id">{{ id }}</option>
</select>
```

`selected` 为数组。

---

## value 绑定类型

```vue
<input type="checkbox" v-model="flag" />        <!-- boolean -->
<input type="number" v-model.number="age" />   <!-- 见修饰符篇 -->
<input v-model="obj" />                         <!-- 少见，通常 string -->
```

---

## 动态 attribute 与 disabled

```vue
<input
  v-model="email"
  :disabled="loading"
  autocomplete="email"
/>
```

表单控件的 **aria-\***、**name** 应保留以利于 a11y 与原生提交（若不用 JS 提交）。

---

## 与 @submit 配合

```vue
<script setup>
const form = ref({ name: '', email: '' })

function onSubmit(e) {
  e.preventDefault()
  api.save(form.value)
}
</script>

<template>
  <form @submit="onSubmit">
    <input v-model="form.name" name="name" />
    <input v-model="form.email" type="email" />
    <button type="submit">保存</button>
  </form>
</template>
```

阻止默认跳转用 **`@submit.prevent`**。

## 常见坑

| 坑 | 修复 |
|----|------|
| select 初始 `''` 无匹配 option | 提供 placeholder option |
| checkbox 数组未初始化 | `ref([])` |
| number input 得到 string | `.number` 修饰符 |
| 中文 IME 组合输入 | 默认 v-model 已处理 composition |

## 小结

**v-model 本质**：`:value`/`:checked` + `@input`/`@change` 语法糖，随控件类型切换绑定属性与事件（见上表）。

**文本**：input/textarea 绑 value + input；textarea 不用子节点插值。

**checkbox**：单个绑 boolean；多个同 v-model 绑数组，靠 value 区分选项。

**radio**：同组共享 v-model，靠 value 决定选中值。

**select**：单选绑 string/number；`multiple` 时绑数组；初始值要有匹配 option。

**提交**：`@submit.prevent` + v-model 字段；保留 name/aria 属性利于 a11y。

**常见坑**：select 空初始值、checkbox 数组未初始化、number 得 string（用 .number）。

**自定义组件** 遵循 modelValue 协议，与原生行为对齐。
