# 自定义表单组件协议

自定义输入组件 v-model = **`modelValue` + `update:modelValue`**，Vue 3.4+ 用 **defineModel** 一行搞定；对象型 model 用 immutable 更新 + emit，勿直接改 props。

---

## 标准协议

```vue
<!-- BaseInput.vue -->
<script setup>
const props = defineProps({
  modelValue: String,
  placeholder: String
})
const emit = defineEmits(['update:modelValue'])

function onInput(e) {
  emit('update:modelValue', e.target.value)
}
</script>

<template>
  <input
    :value="modelValue"
    :placeholder="placeholder"
    @input="onInput"
  />
</template>
```

父组件：

```vue
<BaseInput v-model="keyword" placeholder="搜索" />
```

---

## defineModel（Vue 3.4+）

```vue
<script setup>
const model = defineModel({ type: String, required: true })
</script>

<template>
  <input v-model="model" />
</template>
```

编译器展开为 props + emit，**少样板代码**。

---

## 多个 v-model

```vue
<!-- RangeInput.vue -->
<script setup>
const start = defineModel('start', { type: Number })
const end = defineModel('end', { type: Number })
</script>

<template>
  <input type="number" v-model.number="start" />
  <input type="number" v-model.number="end" />
</template>
```

```vue
<RangeInput v-model:start="range[0]" v-model:end="range[1]" />
```

---

## 非字符串 modelValue

```vue
<script setup>
const model = defineModel({ type: [String, Number, null] })
</script>
```

复杂对象（如选项 `{ label, value }`）也可作为 modelValue，父组件传整个对象。

---

## 包装第三方组件

```vue
<script setup>
import { ElInput } from 'element-plus'

const model = defineModel()
</script>

<template>
  <ElInput v-model="model" clearable />
</template>
```

业务组件层统一 **v-model 对外**，内部适配 UI 库。

---

## checkbox / switch 自定义

```vue
<script setup>
const checked = defineModel({ type: Boolean, default: false })
</script>

<template>
  <button
    type="button"
    role="switch"
    :aria-checked="checked"
    @click="checked = !checked"
  >
    {{ checked ? '开' : '关' }}
  </button>
</template>
```

布尔 modelValue 常见于 Switch、Checkbox 封装。

---

## 表单字段 + 校验状态

```vue
<script setup>
const model = defineModel()
defineProps({
  error: String,
  label: String
})
</script>

<template>
  <label>
    {{ label }}
    <input v-model="model" :aria-invalid="!!error" />
    <span v-if="error" role="alert">{{ error }}</span>
  </label>
</template>
```

校验逻辑可放父级或 vee-validate Field 包裹。

---

## v-model 与 props 单向

子组件**不要**直接改 props 对象深层字段再指望 v-model；应 emit 新值：

```js
// ✅
emit('update:modelValue', { ...props.modelValue, name: v })

// ❌ 父传对象，子改 props.modelValue.name
```

---

## TypeScript

```vue
<script setup lang="ts">
const model = defineModel<string>()
const ids = defineModel<number[]>('ids')
</script>
```

---

## 与原生 name 表单提交

自定义组件默认不在原生 submit 中；若需参与，加 hidden input 或父级统一序列化 `form` state。

---

## 小结

**标准协议**：`modelValue` prop + `update:modelValue` emit；具名 v-model 用对应 prop/emit 名（如 `title` / `update:title`）。

**defineModel**（3.4+）首选：编译展开为 props+emit，支持多 v-model 和修饰符。

**包装 UI 库**：业务组件对外统一 v-model，内部适配 Element Plus 等，便于替换底层库。

**布尔 model**：Switch/Checkbox 封装常用 Boolean modelValue。

**对象 model**：immutable 更新 + emit，禁止直接改 props 深层字段。

**校验展示**：error/label props + aria-invalid/role="alert"；校验逻辑放父或 vee-validate。

**TypeScript**：defineModel 泛型指定类型；具名 model 同理。

**原生 submit**：自定义组件默认不参与；需 hidden input 或父级序列化。
