# 泛型组件与库导出

Select、Table 等数据驱动组件需要类型安全的 `T`。Vue 3.3+ **script setup generic** 让 props 和 v-model 随使用者指定的类型变化；组件库发布用 **vite-plugin-dts** 生成声明并导出 Props 类型。

---

## 为什么需要泛型组件

```vue
<!-- 无泛型：value 只能是 unknown -->
<script setup lang="ts">
defineProps<{ options: Option[]; modelValue: unknown }>();
</script>
```

```vue
<!-- 有泛型：T 由使用者指定 -->
<script setup lang="ts" generic="T extends string | number">
defineProps<{
  options: { label: string; value: T }[];
  modelValue: T;
}>();
</script>
```

父组件传入 `T = number` 时，`modelValue` 与 `update:modelValue` 均为 number。

---

## generic 语法

```vue
<script setup lang="ts" generic="T, U extends T">
defineProps<{
  items: U[];
  selected: T | null;
}>();
</script>
```

| 形式 | 说明 |
|------|------|
| `generic="T"` | 单类型参数 |
| `generic="T extends Record<string, unknown>"` | 带约束 |
| 多参数 | 逗号分隔 |

## GenericSelect 完整示例

```vue
<!-- components/GenericSelect.vue -->
<script setup lang="ts" generic="T extends string | number | boolean">
const props = defineProps<{
  options: { label: string; value: T }[];
  modelValue: T | null;
}>();

const emit = defineEmits<{
  'update:modelValue': [value: T | null];
}>();

function onChange(val: T) {
  emit('update:modelValue', val);
}
</script>

<template>
  <select :value="modelValue ?? ''" @change="onChange(($event.target as HTMLSelectElement).value as T)">
    <option v-for="opt in options" :key="String(opt.value)" :value="opt.value">
      {{ opt.label }}
    </option>
  </select>
</template>
```

```vue
<!-- 使用 -->
<script setup lang="ts">
import GenericSelect from '@/components/GenericSelect.vue';

const status = ref<'draft' | 'published'>('draft');
const options = [
  { label: '草稿', value: 'draft' as const },
  { label: '已发布', value: 'published' as const },
];
</script>

<template>
  <GenericSelect v-model="status" :options="options" />
</template>
```

## 与 defineModel 组合

```vue
<script setup lang="ts" generic="T">
const model = defineModel<T>();
</script>
```

简化双向绑定泛型组件。

## 库导出类型

### 单文件导出

```ts
// components/MyButton/index.ts
export { default as MyButton } from './MyButton.vue';
export type { MyButtonProps } from './MyButton.vue';
```

SFC 中导出 interface：

```vue
<script setup lang="ts">
export interface MyButtonProps {
  type?: 'primary' | 'default';
  loading?: boolean;
}
defineProps<MyButtonProps>();
</script>
```

### vite-plugin-dts

```bash
pnpm add -D vite-plugin-dts
```

```ts
// vite.config.lib.ts
import dts from 'vite-plugin-dts';

export default defineConfig({
  build: { lib: { entry: 'src/index.ts', formats: ['es'] } },
  plugins: [vue(), dts({ insertTypesEntry: true })],
});
```

```json
// package.json
{
  "name": "@my/ui",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "import": "./dist/index.js",
      "types": "./dist/index.d.ts"
    }
  }
}
```

## 消费方体验

```ts
import { MyButton, type MyButtonProps } from '@my/ui';

const props: MyButtonProps = { type: 'primary' };
```

GlobalComponents augmentation（可选）：

```ts
// types/components.d.ts
import type { MyButton } from '@my/ui';

declare module 'vue' {
  export interface GlobalComponents {
    MyButton: typeof MyButton;
  }
}
```

## 泛型组件的限制

| 限制 | 说明 |
|------|------|
| 模板推断 | 有时需显式传 generic |
| HMR | 改 generic 约束可能需全刷 |
| vue-tsc | 需 3.3+ 与新版 vue-tsc |

复杂场景可用 **工厂函数** 替代：

```ts
export function createList<T>() {
  return defineComponent(/* ... */) as DefineComponent<{ items: T[] }>;
}
```

---

## 与 React 对照

| Vue | React |
|-----|-------|
| `generic="T"` SFC | `<T,>(props: Props<T>)` |
| defineProps 泛型 | FC 泛型 |
| defineExpose | forwardRef + imperative |

---

## 小结

**generic SFC**：`<script setup lang="ts" generic="T extends ...">` 参数化 props 和 emit；Select/Table 等数据组件类型安全。

**defineModel + generic**：`defineModel<T>()` 简化泛型 v-model。

**库发布**：`vite-plugin-dts` 生成 `.d.ts`；SFC 内 `export interface XxxProps`；`package.json` exports 指向 types。

**消费方**：import 组件 + Props 类型；可选 GlobalComponents augmentation 全局模板提示。

**限制**：模板推断有时需显式类型；复杂场景可用 `createList<T>()` 工厂。
