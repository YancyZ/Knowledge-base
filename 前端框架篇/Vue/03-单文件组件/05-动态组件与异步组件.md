# 动态组件与异步组件

`<component :is>` 在同一挂载点切换组件；**defineAsyncComponent** 按需拆 chunk，Tab、向导步骤与路由懒加载的常用组合。

---

## 动态组件 component :is

```vue
<template>
  <div class="tabs">
    <button
      v-for="tab in tabs"
      :key="tab.name"
      @click="active = tab.name"
    >
      {{ tab.label }}
    </button>

    <component :is="activeComponent" />
  </div>
</template>

<script setup>
import { computed, ref } from 'vue'
import TabHome from './TabHome.vue'
import TabProfile from './TabProfile.vue'

const tabs = [
  { name: 'home', label: '首页', comp: TabHome },
  { name: 'profile', label: '资料', comp: TabProfile }
]
const active = ref('home')

const activeComponent = computed(() =>
  tabs.find(t => t.name === active.value)?.comp
)
</script>
```

| `:is` 可接受 | 示例 |
|--------------|------|
| 组件对象 | `TabHome` |
| 字符串（已注册或 HTML 标签） | `'div'`、`'MyComp'` |
| 异步组件 | `defineAsyncComponent(...)` |

## KeepAlive 缓存实例

切换 Tab 时默认销毁旧组件；用 **KeepAlive** 保留状态：

```vue
<KeepAlive :include="['TabHome', 'TabProfile']" :max="10">
  <component :is="activeComponent" :key="active" />
</KeepAlive>
```

| prop | 说明 |
|------|------|
| `include` / `exclude` | 组件 **name** 匹配（字符串、数组、正则） |
| `max` | LRU 缓存上限 |

被缓存组件会触发 **`activated` / `deactivated`** 钩子。`:key` 强制同一 is 不同实例时区分。

## defineAsyncComponent

**基础懒加载**：

```javascript
import { defineAsyncComponent } from 'vue'

const AsyncChart = defineAsyncComponent(() =>
  import('./HeavyChart.vue')
)
```

```vue
<template>
  <AsyncChart v-if="showChart" />
</template>
```

构建工具将 `./HeavyChart.vue` 打成单独 JS 文件，首次渲染时 fetch。

**加载与错误状态**：

```javascript
const AsyncPanel = defineAsyncComponent({
  loader: () => import('./Panel.vue'),
  loadingComponent: LoadingSpinner,
  errorComponent: ErrorDisplay,
  delay: 200,
  timeout: 10000,
  suspensible: false
})
```

| 选项 | 作用 |
|------|------|
| `loader` | 返回 Promise 的 import |
| `loadingComponent` | 加载中占位 |
| `errorComponent` | 失败 UI |
| `delay` | 延迟显示 loading，避免闪烁 |
| `timeout` | 超时 reject |

**与 Suspense**，异步 setup 或 async 组件可配合 **Suspense** 统一 fallback：

```vue
<Suspense>
  <AsyncPage />
  <template #fallback>加载页面…</template>
</Suspense>
```

## 路由懒加载（对照）

```javascript
{
  path: '/report',
  component: () => import('@/views/ReportView.vue')
}
```

路由级 lazy 与 `defineAsyncComponent` 都产生 code splitting；路由场景通常 **直接 import()** 即可，不必再包一层 async component。

```mermaid
flowchart LR
  U[用户导航]
  U --> R[Router 匹配]
  R --> I[import chunk]
  I --> C[挂载组件]
```

## shallowRef 与动态组件

`:is` 绑定组件对象时，若组件引用不变但内部变，用 **shallowRef** 存当前组件，避免深层响应式开销：

```javascript
import { shallowRef } from 'vue'
const current = shallowRef(TabHome)

function switchTo(comp) {
  current.value = comp
}
```

## 动态 + 权限

```vue
<script setup>
import { computed } from 'vue'
import AdminPanel from './AdminPanel.vue'
import GuestPanel from './GuestPanel.vue'

const props = defineProps(['role'])
const panel = computed(() =>
  props.role === 'admin' ? AdminPanel : GuestPanel
)
</script>

<template>
  <component :is="panel" />
</template>
```

复杂权限表可映射 **name → 组件** 字典，未授权组件不 import（配合路由守卫更常见）。

## Vue 2 差异

| 点 | Vue 2 | Vue 3 |
|----|-------|-------|
| 异步组件 | `() => import()` 或对象形式 | `defineAsyncComponent` 统一 |
| KeepAlive | `include` 依赖组件 name | 同左；script setup 需额外 script 设 name |
| functional + :is | 常见 | 函数组件简化 |

---

## 常见坑

| 坑 | 处理 |
|----|------|
| KeepAlive 不生效 | 检查组件 **name** 与 include 字符串 |
| 切换闪屏 | loading delay、Suspense、骨架屏 |
| 同一 is 状态串 | 加 **:key** |
| 循环依赖 chunk | 调整 import 方向或抽公共模块 |
| SSR 异步组件 | 需 Nuxt/SSR 配置配合，避免 hydration mismatch |

---

## 选型建议

| 需求 | 方案 |
|------|------|
| Tab 切换 | `:is` + 可选 KeepAlive |
| 大对话框首次打开 | defineAsyncComponent |
| 整页路由 | Router lazy import |
| 条件渲染二选一 | `v-if` 两组件往往更简单 |

---

## 小结

要点：`<component :is>` 在同一 DOM 挂载点切换不同组件；defineAsyncComponent 把组件打成独立 chunk 按需加载；KeepAlive 缓存已卸载的组件实例以保留状态。


- `<component :is>`：同一挂载点切换组件；配 **`:key`** 强制重建实例。
- KeepAlive：缓存动态组件状态；`include`/`exclude` 按 name 控制。
- defineAsyncComponent：独立 chunk + loading/error 组件；路由级拆包用 `() => import()`。
- shallowRef 存组件引用，避免深层响应式开销。

**易混点**：
- KeepAlive 的 include 匹配的是组件 **name**，script setup 需 defineOptions 设 name。
- 路由 lazy import 不必再包 defineAsyncComponent。
- 同一 `:is` 不同实例需加 `:key` 防状态串。

核对：动态组件是否设了 name？KeepAlive include 字符串是否匹配？大组件是否做了懒加载？
