# 生命周期组合式对照

Composition API 用 **`onMounted`/`onUnmounted`** 等注册生命周期，与 Options 钩子一一对应，可在 composable 内调用（须 setup 顶层同步注册）。

---

## 对照表

| Options API | Composition API |
|-------------|-----------------|
| beforeCreate | 无（逻辑直接放 setup 顶层） |
| created | 无（setup 本身等价 created 之前段） |
| beforeMount | **onBeforeMount** |
| mounted | **onMounted** |
| beforeUpdate | **onBeforeUpdate** |
| updated | **onUpdated** |
| beforeUnmount | **onBeforeUnmount** |
| unmounted | **onUnmounted** |
| activated | **onActivated** |
| deactivated | **onDeactivated** |
| errorCaptured | **onErrorCaptured** |
| renderTracked | **onRenderTracked**（dev） |
| renderTriggered | **onRenderTriggered**（dev） |

```javascript
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted
} from 'vue'
```

---

## 基本用法

```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const list = ref([])

onMounted(async () => {
  list.value = await fetchList()
  window.addEventListener('focus', refresh)
})

onUnmounted(() => {
  window.removeEventListener('focus', refresh)
})

async function refresh() {
  list.value = await fetchList()
}
</script>
```

**beforeCreate / created** 的逻辑：

| 原 created 用途 | Composition |
|-----------------|-------------|
| 初始化非响应式变量 | setup 顶层 |
| 读 props 调 API | setup 顶层 + async |
| 注册 watch | setup 顶层 watch |

---

## 在 composable 中注册

```javascript
export function useInterval(fn, ms) {
  onMounted(() => {
    const id = setInterval(fn, ms)
    onUnmounted(() => clearInterval(id))
  })
}
```

onUnmounted 嵌套在 onMounted 内注册合法，卸载时仍会执行。

---

## 执行顺序（与 Options 一致）

**挂载**：父 beforeMount → 子 beforeMount → 子 onMounted → 父 onMounted。

**卸载**：子 onBeforeUnmount → 父 onBeforeUnmount → 子 onUnmounted → 父 onUnmounted。

**与 Options 混写**，两者都会执行；避免重复请求。

---

## KeepAlive

```vue
<script setup>
import { onActivated, onDeactivated } from 'vue'

onActivated(() => {
  resumePolling()
})

onDeactivated(() => {
  pausePolling()
})
</script>
```

Tab 切换缓存时 **onDeactivated** 替代 onUnmounted 暂停，**onActivated** 恢复。

---

## onErrorCaptured

```javascript
import { onErrorCaptured, ref } from 'vue'

const err = ref(null)

onErrorCaptured((e, instance, info) => {
  err.value = e
  console.warn(info)
  return false // 阻止继续向上传播
})
```

捕获后代组件渲染/侦听器错误；返回 false 阻止冒泡。

## 开发调试钩子

```javascript
import { onRenderTracked, onRenderTriggered } from 'vue'

onRenderTracked(({ key, target, type }) => {
  console.log('track', key)
})
```

仅开发环境有意义，用于排查意外依赖。

## SSR 注意

| 钩子 | SSR |
|------|-----|
| setup | 每请求执行 |
| onMounted 及之后 DOM 相关 | **不**在服务端执行 |
| onUnmounted | 不执行（无挂载） |

服务端数据预取用 **onServerPrefetch** 或 Nuxt 的 **useAsyncData**。

```javascript
import { onServerPrefetch } from 'vue'

onServerPrefetch(async () => {
  await store.fetchData()
})
```

---

## 调用规则

| 规则 | 说明 |
|------|------|
| 同步注册 | onMounted 须在 setup 同步执行流里调用 |
| 勿在 await 后注册 | async setup 中 await 之后的 onMounted 会报警告 |
| 循环/条件 | 避免；导致钩子顺序不确定 |

```javascript
// ❌
setup async () {
  await delay()
  onMounted(() => {}) // 太迟
}

// ✅
setup() {
  onMounted(() => {})
  ;(async () => { await load() })()
}
```

## 迁移清单

| Options | 动作 |
|---------|------|
| created + mounted 都有 fetch | 合并到 onMounted 或拆 composable |
| destroyed | onUnmounted |
| beforeDestroy 清理 | onBeforeUnmount |
| activated | onActivated |

---

## 小结

要点：Composition API 用 onXxx 函数注册生命周期回调，与 Options 钩子一一对应；setup 顶层替代 beforeCreate/created，须在同步执行流里注册。


- setup 顶层替代 beforeCreate/created；DOM 操作与订阅用 onMounted/onUnmounted。
- composable 内可注册 onXxx，与调用组件生命周期绑定。
- KeepAlive：onActivated/onDeactivated；SSR 跳过仅客户端钩子。
- 规则：钩子须 setup 顶层**同步**注册；勿在 async 回调后再注册。

**易混点**：
- Options mounted 与 setup onMounted 并存时两者都执行。
- async setup 里 await 后注册 onMounted 无效。
- onMounted 在 SSR 不执行，服务端数据用 onServerPrefetch。

核对：钩子是否在 setup 顶层同步注册？mounted 副作用是否在 onUnmounted 清理？KeepAlive 组件是否用了 onActivated？
