# 性能调优

Vue 性能调优先测量再动手：减渲染（v-memo/shallow）、减包体（懒加载）、减 DOM（虚拟列表）；KeepAlive 记得设 max。

## 性能优化流程

```mermaid
flowchart LR
  Measure[测量 RUM/Lighthouse] --> Find[定位瓶颈]
  Find --> Fix[针对性优化]
  Fix --> Measure
```

**勿过早优化**：先数据证明 LCP/INP 问题，再改代码。

## 减少不必要渲染

| 手段 | 场景 |
|------|------|
| `computed` 缓存 | 衍生数据 |
| `shallowRef` / `shallowReactive` | 大对象仅需替换引用 |
| `v-once` | 纯静态子树 |
| `v-memo` | 昂贵列表项，依赖不变则跳过 |
| `defineOptions({ inheritAttrs: false })` | 减少 attrs 传递 |

```vue
<div v-for="item in list" :key="item.id" v-memo="[item.selected, item.name]">
  <ExpensiveRow :item="item" />
</div>
```

## KeepAlive 缓存

```vue
<RouterView v-slot="{ Component }">
  <KeepAlive :include="['UserList', 'Dashboard']" :max="10">
    <component :is="Component" :key="$route.fullPath" />
  </KeepAlive>
</RouterView>
```

| 参数 | 说明 |
|------|------|
| `include` | 组件 `name` 白名单 |
| `max` | LRU 上限，防内存泄漏 |
| `activated` / `deactivated` | 刷新数据时机 |

## 长列表：虚拟滚动

```bash
pnpm add @tanstack/vue-virtual
```

```vue
<script setup lang="ts">
import { useVirtualizer } from '@tanstack/vue-virtual';

const parentRef = ref<HTMLElement>();
const rows = ref(Array.from({ length: 10000 }, (_, i) => i));

const virtualizer = useVirtualizer({
  count: rows.value.length,
  getScrollElement: () => parentRef.value,
  estimateSize: () => 48,
});
</script>

<template>
  <div ref="parentRef" style="height: 400px; overflow: auto">
    <div :style="{ height: `${virtualizer.getTotalSize()}px`, position: 'relative' }">
      <div
        v-for="vRow in virtualizer.getVirtualItems()"
        :key="vRow.key"
        :style="{
          position: 'absolute',
          top: 0,
          transform: `translateY(${vRow.start}px)`,
          height: `${vRow.size}px`,
        }"
      >
        Row {{ rows[vRow.index] }}
      </div>
    </div>
  </div>
</template>
```

万级 DOM 勿裸 `v-for`。

## 响应式边界

```ts
import { shallowRef, markRaw } from 'vue';

// 第三方实例不需深度响应式
const chart = shallowRef(markRaw(echarts.init(el)));
```

| API | 用途 |
|-----|------|
| `shallowRef` | 仅 `.value` 替换触发更新 |
| `markRaw` | 永不代理 |
| `readonly` | 防子组件误改 |

## 反模式

| 反模式 | 后果 |
|--------|------|
| 巨大 reactive 树 | 任意字段变更触发大范围更新 |
| watch 一切 | CPU 浪费 |
| 无 key 的 v-for | 错误复用 DOM |
| 过度 KeepAlive | 内存涨 |

---

## 小结

Vue 性能调优先测量再动手：DevTools / Lighthouse 定位瓶颈，针对性优化后再验证。渲染侧用 `v-memo`、`shallowRef` 减少不必要更新；包体侧用路由懒加载与 UI 库按需引入；万级列表用虚拟滚动减 DOM 节点。KeepAlive 设 `max` 防内存泄漏。SSR/Nuxt 注意关键 CSS 内联与 `ClientOnly` 包裹仅客户端组件。勿过早优化，先数据证明 LCP/INP 问题再改代码。
