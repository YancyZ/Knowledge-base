## 三、例题：React 渲染流程？

**定义**：触发更新后经历 render（可中断）与 commit（改 DOM）。

**机制**：
1. setState / 父 render 等调度更新  
2. render 阶段执行组件、Diff  
3. commit 改 DOM、跑 layout effect  
4. 浏览器绘制后 passive effect  

**例子**：`setCount` → 函数重跑 → 文本节点更新。

**权衡**：为何 Concurrent 要可中断？避免长任务挡输入。

见 [06-01](../06-渲染与调和/01-渲染流程总览.md)。

## 四、例题：useEffect 与 useLayoutEffect？

| | useEffect | useLayoutEffect |
|---|-----------|-----------------|
| 时机 | paint 后 | paint 前 |
| 阻塞绘制 | 否 | 是 |
| 场景 | 拉数据、订阅 | 测 DOM、同步改 UI |

**别滥用 layout**——默认 useEffect。

## 五、例题：如何优化 React 性能？

**分层答**：

1. **加载**：路由 lazy、减包  
2. **渲染次数**：状态下沉、memo、Context 拆分  
3. **渲染重量**：虚拟列表  
4. **交互**：startTransition  
5. **测量**：Profiler、web-vitals  

见 [11-06 Checklist](../11-性能优化/06-性能优化Checklist.md)。

## 六、例题：服务端状态放哪？

**定义**：API 数据是服务端真相的缓存副本。

**答**：TanStack Query / SWR；不放 Redux 除非已有投资。

**对比**：客户端 UI 状态 → Zustand / useState。

见 [08-01](../08-状态管理/01-状态分类与放置原则.md)。

## 七、例题：key 的作用？

**答**：帮助 Diff 识别**同一层级**节点身份，重排时复用 DOM。

**反例**：index key 在重排、插入时导致状态错乱。
