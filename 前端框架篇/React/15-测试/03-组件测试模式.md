# 组件测试模式

> 表单、列表、Modal、异步加载——常见 UI 有**可复用测试模式**。本篇按场景给模板，直接套用再改业务断言。

---

## 一、表单提交

```tsx
it('校验失败显示错误', async () => {
  const user = userEvent.setup();
  const onSubmit = vi.fn();
  render(<LoginForm onSubmit={onSubmit} />);

  await user.click(screen.getByRole('button', { name: '登录' }));
  expect(await screen.findByText('请输入邮箱')).toBeInTheDocument();
  expect(onSubmit).not.toHaveBeenCalled();
});

it('提交成功', async () => {
  const user = userEvent.setup();
  const onSubmit = vi.fn();
  render(<LoginForm onSubmit={onSubmit} />);

  await user.type(screen.getByLabelText('邮箱'), 'a@b.com');
  await user.type(screen.getByLabelText('密码'), 'secret');
  await user.click(screen.getByRole('button', { name: '登录' }));

  await waitFor(() => expect(onSubmit).toHaveBeenCalledWith({
    email: 'a@b.com',
    password: 'secret',
  }));
});
```

| 点 | 说明 |
|----|------|
| `vi.fn()` | Vitest mock |
| label 关联 | getByLabelText |

## 二、异步 loading / 空态 / 错误

```tsx
it('加载后显示列表', async () => {
  vi.mocked(fetchUsers).mockResolvedValue([{ id: '1', name: 'Li' }]);
  render(<UserList />);

  expect(screen.getByText(/加载/)).toBeInTheDocument();
  expect(await screen.findByText('Li')).toBeInTheDocument();
});

it('空列表', async () => {
  vi.mocked(fetchUsers).mockResolvedValue([]);
  render(<UserList />);
  expect(await screen.findByText('暂无数据')).toBeInTheDocument();
});

it('错误态可重试', async () => {
  vi.mocked(fetchUsers).mockRejectedValue(new Error('fail'));
  render(<UserList />);
  await user.click(await screen.findByRole('button', { name: '重试' }));
  expect(fetchUsers).toHaveBeenCalledTimes(2);
});
```

## 三、Modal / Dialog

```tsx
it('打开关闭对话框', async () => {
  const user = userEvent.setup();
  render(<App />);

  await user.click(screen.getByRole('button', { name: '删除' }));
  const dialog = screen.getByRole('dialog');
  expect(within(dialog).getByText('确认删除？')).toBeInTheDocument();

  await user.click(within(dialog).getByRole('button', { name: '取消' }));
  expect(screen.queryByRole('dialog')).not.toBeInTheDocument();
});
```

配合 `@testing-library/user-event` 的 `{ skipHover: true }` 等按需。

## 四、列表与筛选

```tsx
it('筛选结果', async () => {
  render(<ProductList products={mockProducts} />);
  await user.selectOptions(screen.getByLabelText('分类'), 'electronics');
  expect(screen.getByText('手机')).toBeInTheDocument();
  expect(screen.queryByText('沙发')).not.toBeInTheDocument();
});
```

## 五、Mock 模块

```tsx
vi.mock('../api/users', () => ({
  fetchUsers: vi.fn(),
}));

import { fetchUsers } from '../api/users';
```

| 层级 | mock 谁 |
|------|---------|
| 组件测试 | API 模块 |
| Hook 测试 | fetch 或依赖 inject |

## 六、MSW（可选）

用 **Mock Service Worker** 拦截网络，更接近集成：

```tsx
import { setupServer } from 'msw/node';
import { http, HttpResponse } from 'msw';

const server = setupServer(
  http.get('/api/users', () => HttpResponse.json([{ id: '1', name: 'Li' }])),
);

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());
```
