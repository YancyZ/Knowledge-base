# 组件库类型导出

做 **UI 组件库** 或 **内部 design system** 时，类型导出与 `package.json exports` 同样重要，消费者靠 `.d.ts` 获得智能提示。

---

## 包结构

```
my-ui/
├── src/
│   ├── Button/
│   │   ├── Button.tsx
│   │   └── index.ts
│   └── index.ts
├── dist/
│   ├── index.js
│   └── index.d.ts
└── package.json
```

```json
{
  "name": "@acme/ui",
  "main": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js"
    }
  }
}
```

`types` 字段和 `exports.types` 都要配置，确保消费方能解析到 `.d.ts`。

---

## barrel export

```tsx
// src/index.ts
export { Button } from './Button';
export type { ButtonProps } from './Button';

export { ThemeProvider, useTheme } from './theme';
export type { Theme } from './theme';
```

| 实践 | 说明 |
|------|------|
| `export type` | 仅类型，运行时无开销 |
| 避免 `export *` 过度 | 控制公共 API 面 |

barrel 文件集中导出公共 API，用 `export type` 分离类型导出，避免 `export *` 暴露内部实现。

---

## Props 导出命名

```tsx
// Button.tsx
export interface ButtonProps extends React.ComponentPropsWithoutRef<'button'> {
  variant?: 'primary' | 'secondary';
  size?: 'sm' | 'md' | 'lg';
}

export function Button(props: ButtonProps) { ... }
```

消费者：

```tsx
import { Button, type ButtonProps } from '@acme/ui';

const props: ButtonProps = { variant: 'primary', children: 'OK' };
```

Props 接口命名 `ComponentNameProps`，与组件同文件 export，消费方用 `import type` 引入。

---

## Variant 类型共享

```tsx
export const buttonVariants = ['primary', 'secondary', 'ghost'] as const;
export type ButtonVariant = (typeof buttonVariants)[number];
```

与 CVA（class-variance-authority）配合：

```tsx
import { cva, type VariantProps } from 'class-variance-authority';

const button = cva('btn', {
  variants: {
    variant: { primary: 'btn-primary', secondary: 'btn-secondary' },
    size: { sm: 'btn-sm', md: 'btn-md' },
  },
  defaultVariants: { variant: 'primary', size: 'md' },
});

export type ButtonProps = React.ComponentPropsWithoutRef<'button'> &
  VariantProps<typeof button>;
```

`as const` + 索引类型导出 variant 联合；CVA 的 `VariantProps` 自动从 cva 定义推导 props 类型。

## Polymorphic `as` 类型

```tsx
type AsProp<C extends React.ElementType> = { as?: C };

type PropsToOmit<C extends React.ElementType, P> = keyof (AsProp<C> & P);

type PolymorphicComponentProp<
  C extends React.ElementType,
  Props = {},
> = React.PropsWithChildren<Props & AsProp<C>> &
  Omit<React.ComponentPropsWithoutRef<C>, PropsToOmit<C, Props>>;

export function Box<C extends React.ElementType = 'div'>({
  as,
  ...props
}: PolymorphicComponentProp<C, { className?: string }>) {
  const Component = as ?? 'div';
  return <Component {...props} />;
}
```

复杂但 shadcn / Radix 同类组件常用。`as` prop 让组件渲染为不同 HTML 元素，同时继承对应元素属性。

## 构建与 tsc

| 工具 | 说明 |
|------|------|
| **tsup** | 打包 + d.ts 一键 |
| **vite library mode** | 库模式 |
| `declaration: true` | tsc 出类型 |

```json
// tsconfig.build.json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "declaration": true,
    "emitDeclarationOnly": true,
    "outDir": "dist"
  },
  "include": ["src"]
}
```

tsup 或 tsc `declaration: true` 生成 `.d.ts`，与 JS 产物一起发布。

## peerDependencies

```json
{
  "peerDependencies": {
    "react": "^18.0.0 || ^19.0.0",
    "react-dom": "^18.0.0 || ^19.0.0"
  },
  "devDependencies": {
    "@types/react": "^18.3.0",
    "react": "^18.3.0"
  }
}
```

库不打包 React，类型对齐 consumer 的 `@types/react`。React 作为 peerDependency，避免多实例。

## Breaking 变更

| 变更 | 影响 |
|------|------|
| 删 export | major bump |
| props 必填化 | major |
| 可选 → 更窄联合 | 可能 major |

用 **Changesets** 或 semver 管理。Props 类型变更影响消费者编译，应按 semver 规范 bump 版本。

---

## 小结

组件库须 export type Props、配置 package.json types 字段、peer react 不打包进 bundle。

组件库类型导出要点：package.json 配置 `types` 和 `exports.types` 指向 `.d.ts`；barrel 用 `export type { ButtonProps }` 控制公共 API。Props 命名 `ComponentNameProps`，与组件同文件 export。Variant 用 `as const` + 索引类型或 CVA `VariantProps` 共享。Polymorphic `as` 类型较复杂，Radix/shadcn 同类组件常用。构建用 tsup 或 tsc declaration 生成类型文件。React 放 peerDependencies 不打包进 bundle。Breaking 变更（删 export、props 必填化）应 major bump，用 Changesets 管理版本。
