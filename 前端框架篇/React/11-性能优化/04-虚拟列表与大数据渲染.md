# 虚拟列表与大数据渲染

渲染 **1 万行 DOM** 即使用户只看 20 行，**虚拟列表（窗口化）** 只 mount 可视区 + 缓冲区的项，滚动时复用节点。

---

## 问题本质

```mermaid
flowchart TB
  subgraph bad [全量渲染]
    D1[10000 个 DOM 行]
  end
  subgraph good [虚拟列表]
    V[可视 ~15 行 + buffer]
    V --> Pool[节点复用]
  end
```

| 指标 | 1 万行全渲染 | 虚拟列表 |
|------|--------------|----------|
| DOM 节点 | ~1 万 | ~30 |
| 首次 paint | 慢 | 快 |
| 滚动 | 易掉帧 | 流畅 |

全量渲染的问题不在 React diff 本身，而在浏览器要维护和管理海量 DOM 节点。虚拟列表只渲染视口内可见的行加上少量缓冲，滚动时复用节点、更新内容。

---

## 原理

```
┌─────────────────────────────┐  ← 容器 viewport
│  [padding-top 占位]          │
│  ┌ row 100 ┐                │
│  ┌ row 101 ┐  可见区         │
│  ┌ row 102 ┐                │
│  [padding-bottom 占位]       │
└─────────────────────────────┘
总高度 = itemCount × itemHeight（或动态测量）
```

| 概念 | 说明 |
|------|------|
| **scrollTop** | 决定起始 index |
| **itemHeight** | 固定高最简单 |
| **overscan** | 上下多渲染几行防白屏 |

上下用 padding 占位撑开滚动条，中间只渲染可见项。`scrollTop` 变化时重新计算起始 index，绝对定位或 transform 放置每一行。

## @tanstack/react-virtual

```bash
pnpm add @tanstack/react-virtual
```

```tsx
import { useVirtualizer } from '@tanstack/react-virtual';
import { useRef } from 'react';

function VirtualList({ rows }: { rows: string[] }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const virtualizer = useVirtualizer({
    count: rows.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 36,
    overscan: 5,
  });

  return (
    <div ref={parentRef} style={{ height: 400, overflow: 'auto' }}>
      <div
        style={{
          height: virtualizer.getTotalSize(),
          position: 'relative',
        }}
      >
        {virtualizer.getVirtualItems().map(vRow => (
          <div
            key={vRow.key}
            style={{
              position: 'absolute',
              top: 0,
              transform: `translateY(${vRow.start}px)`,
              height: vRow.size,
            }}
          >
            {rows[vRow.index]}
          </div>
        ))}
      </div>
    </div>
  );
}
```

TanStack Virtual 是框架无关的虚拟化库，与 React 配合只需 `useVirtualizer` 和滚动容器 ref。

## 固定高 vs 动态高

| 固定高 | 动态高 |
|--------|--------|
| `estimateSize: () => 36` | `measureElement` 实测 |
| 实现简单 | 滚动条跳动需处理 |
| 表格行、统一列表项 | 评论、多行文本 |

固定行高实现最简单，总高度可精确计算。动态高度需要测量 DOM 并在滚动时更新，处理不好会出现滚动条跳动。

## 与表格组件

| 方案 | 说明 |
|------|------|
| **TanStack Table + Virtual** |  head 固定 + body 虚拟 |
| **AG Grid / MUI DataGrid** | 内置虚拟化 |
| 自研 | 仅数据极大时 |

中后台 500 行以内可先不虚拟；**万级**再引入。复杂 colspan 表格优先用成熟表格库，自研虚拟化成本高。

## 其他大数据策略

| 策略 | 场景 |
|------|------|
| **分页** | 服务端分页，最简单 |
| **无限滚动** | 社交 feed，配合 Query 缓存 |
| **Web Worker** | 排序/过滤在 worker |
| **Canvas** | 超密点图、非 DOM |

虚拟化不是唯一方案。能分页就先分页；feed 类场景用无限滚动分批加载；排序过滤极重时可丢到 Worker。

---

## key 与选中态

虚拟列表**复用 DOM**，行内 state 可能错位：

| 问题 | 解决 |
|------|------|
| 输入框内容串行 | 状态放 data 层，不留在 DOM |
| 选中行 | 用 row id 存 Set |

同一 DOM 节点滚动后会展示不同 index 的数据，行内 `useState` 可能保留上一行的值。选中态、编辑态应绑定 row id 而非组件本地 state。

---

## 反模式

| ❌ | ✅ |
|----|-----|
| 100 行就上虚拟 | 先分页 |
| 虚拟 + 复杂 colspan 表格 | 用成熟表格库 |
| 忘记 overscan | 快速滚动白屏 |

---

## 小结

万级 DOM 先考虑分页；确实需要时用 TanStack Virtual 窗口化渲染，注意 overscan 和行内 state 错位。

全量渲染万行 DOM 会导致首次 paint 慢、滚动掉帧；虚拟列表只 mount 视口 + buffer 的行，用 padding 占位和 scrollTop 计算可见范围。TanStack Virtual 是通用实现，固定行高最简单，动态高需 measureElement 并处理滚动条跳动。表格场景可用 TanStack Table + Virtual 或 AG Grid 等内置方案；500 行以内通常不必虚拟。替代策略包括分页、无限滚动、Web Worker 和 Canvas。虚拟列表复用 DOM 时，行内 state 和选中态应绑定 row id，避免内容串行；别忘了 overscan 防止快速滚动白屏。
