# 键盘导航与焦点管理

不少用户**只用键盘**（或 Switch 设备模拟键盘）。Tab 顺序合理、焦点可见、Modal 不「困住」焦点，是 a11y 的核心体验之一。

---

## 键盘基础

```mermaid
flowchart LR
  Tab[Tab / Shift+Tab]
  Enter[Enter / Space]
  Esc[Escape]
  Tab --> Focus[移动焦点]
  Enter --> Activate[激活按钮/链接]
  Esc --> Close[关闭弹层]
```

| 键 | 典型行为 |
|----|----------|
| **Tab** | 下一个可聚焦元素 |
| **Shift+Tab** | 上一个 |
| **Enter / Space** | 按钮、checkbox |
| **Esc** | 关闭 dialog、dropdown |
| **方向键** | 菜单、tabs、radio 组（看组件规范） |

所有鼠标能完成的操作，键盘也应能完成。

---

## 可聚焦元素

| 默认可 focus | 需 tabindex |
|--------------|-------------|
| button、a[href]、input、select、textarea | 自定义 div 按钮（不推荐） |

```tsx
// ❌ div 无键盘激活
<div onClick={onClick}>点击</div>

// ✅
<button type="button" onClick={onClick}>点击</button>
```

---

## tabIndex

| 值 | 含义 |
|----|------|
| `0` | 按文档顺序参与 Tab |
| `-1` | 可 programmatic focus，Tab 跳过 |
| `>0` | **避免**，破坏自然顺序 |

```tsx
// 错误恢复后把焦点移到标题
headingRef.current?.focus(); // 元素需 tabIndex={-1}
```

tabIndex > 0 会破坏自然 Tab 顺序，应通过调整 DOM 顺序解决。

---

## 焦点样式

```css
/* 勿全局 outline: none 不设替代 */
:focus-visible {
  outline: 2px solid var(--focus-ring);
  outline-offset: 2px;
}
```

| ❌ | ✅ |
|----|-----|
| `outline: none` 无替代 | `:focus-visible` 仅键盘显示 |
| 仅 hover 样式 | focus 同样清晰 |

## Modal 焦点陷阱（Focus Trap）

打开 Dialog 时：

1. 焦点移入 Dialog  
2. Tab 只在 Dialog 内循环  
3. 关闭后焦点回到触发按钮  

```tsx
import { Dialog } from '@radix-ui/react-dialog';
// 或 @headlessui/react — 内置 focus trap
```

手写时可查 **focus-trap-react** 或 WAI-ARIA Dialog 模式。

```mermaid
sequenceDiagram
  participant B as 触发按钮
  participant D as Dialog
  B->>D: 打开，focus 进 D
  Note over D: Tab 循环于 D 内
  D->>B: 关闭，focus 回 B
```

## 跳过链接（Skip Link）

```tsx
<a href="#main-content" className="skip-link">
  跳到主内容
</a>
<main id="main-content" tabIndex={-1}>...</main>
```

首屏 Tab 一次直达主内容，少扫整页导航。

---

## React 中的坑

| 坑 | 处理 |
|----|------|
| 条件渲染丢焦点 | 关闭时 `triggerRef.focus()` |
| autoFocus 滥用 | 仅 Modal 首项或明确场景 |
| 路由切换 | 新页 `h1` focus 或 announce |
| 下拉未关 | Esc + 点击外部 |

## 测试

```tsx
await user.tab();
expect(screen.getByRole('button', { name: '下一步' })).toHaveFocus();
await user.keyboard('{Enter}');
```

RTL + userEvent 可测焦点链。

---

## 小结

全功能键盘可达、focus 可见、Modal 焦点陷阱并归还，Tab 顺序勿用 tabindex>0。

键盘用户依赖 Tab/Enter/Esc/方向键完成操作。可聚焦元素优先用 button/a/input 等原生元素，避免 div onClick。tabIndex：0 参与 Tab、-1 仅 programmatic focus、>0 避免。焦点样式用 :focus-visible，勿全局 outline:none。Modal 需 focus trap：打开移入、Tab 循环、关闭归还触发按钮；可用 Radix/Headless UI。Skip Link 跳主内容。React 坑：条件渲染丢焦点、autoFocus 滥用、路由切换 announce。RTL user.tab() 可测焦点链。
