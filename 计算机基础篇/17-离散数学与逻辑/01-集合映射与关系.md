# 集合、映射与关系

**集合**给出「元素归属」的语义，**映射**描述输入到输出的对应，**关系**刻画元素间的有序或等价结构。前端状态建模、权限判定、图遍历与数据库键值，底层都可归结为这三类抽象。

---

## 集合基础

| 概念 | 符号/写法 | 含义 |
|------|-----------|------|
| 属于 | `x ∈ A` | x 是 A 的元素 |
| 子集 | `A ⊆ B` | A 中每个元素都在 B 中 |
| 真子集 | `A ⊂ B` | 子集且不相等 |
| 幂集 | `P(A)` | A 的所有子集构成的集合 |

```javascript
// Set 去重 — 对应数学集合（无序、无重复）
const tags = new Set(['react', 'vue', 'react']);
tags.size; // 2

// 子集判定
function isSubset(a, b) {
  return [...a].every(x => b.has(x));
}
```

**常用运算**：

| 运算 | 含义 | JS 类比 |
|------|------|---------|
| `A ∪ B` | 并 | `[...new Set([...a, ...b])]` |
| `A ∩ B` | 交 | `a.filter(x => b.has(x))` |
| `A \ B` | 差 | `a.filter(x => !b.has(x))` |
| `A × B` | 笛卡尔积 | 路由 `(path, method)` 组合枚举 |

---

## 笛卡尔积与枚举

`|A×B| = |A|·|B|` — 乘法原理的直接推论。REST 路由若 12 条 path × 4 种 method，最多 48 种 `(path, method)` 对；权限矩阵 `Roles × Resources × Actions` 同理。

```javascript
function cartesian(a, b) {
  return a.flatMap(x => b.map(y => [x, y]));
}
// cartesian(['/api', '/admin'], ['GET', 'POST'])
// → [['/api','GET'], ['/api','POST'], ...]
```

**易混点**：笛卡尔积元素是**有序对** `(a,b)`，`(a,b) ≠ (b,a)`（除非 a=b）；集合 `{a,b}` 无序。

## 映射（函数）

映射 `f: A → B` 要求 **A 中每个元素有且仅有一个** B 中像。

| 性质 | 定义 | 前端例子 |
|------|------|----------|
| **单射** | 不同原像 → 不同像 | `userId → profile` 一对一 |
| **满射** | B 中每个元素都有原像 | 枚举值全覆盖的 status 映射 |
| **双射** | 单射 + 满射 | 可逆编码 `id ↔ slug` |

```mermaid
flowchart LR
  A[定义域 A]
  B[陪域 B]
  A -->|f| B
```

**值域 vs 陪域**：值域 `f(A) = { f(x) | x∈A }` 是 B 的子集；陪域是声明的目标集合 B。`status: 0|1|2 → label` 若 label 集合含 `'unknown'` 但从未映射到，则不满射。

```typescript
// Record<K,V> 更接近「全定义映射」
type Status = 0 | 1 | 2;
const label: Record<Status, string> = { 0: 'pending', 1: 'ok', 2: 'fail' };
```

**易混点**：JS 函数可以 `return undefined` 多次、参数可忽略 — 数学映射要求全域定义。

---

## 映射复合

若 `f: A→B`、`g: B→C`，则复合 `g∘f: A→C` 满足 `(g∘f)(x) = g(f(x))`。Redux 中 `selector = memoize(state => view)` 可视为 `state → view` 的映射链；Middleware 则是 `dispatch' = compose(mw)(dispatch)`。

| 性质 | 条件 |
|------|------|
| 复合保持单射 | f、g 均单射 |
| 复合保持满射 | f、g 均满射 |
| 复合一般**不**交换 | `g∘f ≠ f∘g` |

## 关系

二元关系 `R ⊆ A × B`：有序对的集合。

| 关系类型 | 在 `A` 上的性质 | 典型场景 |
|----------|-----------------|----------|
| **等价关系** | 自反、对称、传递 | 同构组件、权限角色分组 |
| **偏序** | 自反、反对称、传递 | 任务依赖 DAG、CSS 层叠优先级 |
| **全序** | 偏序 + 任意两元素可比 | 排序键、z-index 全序（理想情况） |

**等价类**：若 `~` 为等价关系，则 `a` 的等价类 `[a] = { x | x ~ a }`，划分集合 A。

```javascript
// 按「同域名」划分 Cookie 桶 — 等价类思想
function bucketByDomain(cookies) {
  const map = new Map();
  for (const c of cookies) {
    const key = c.domain;
    if (!map.has(key)) map.set(key, []);
    map.get(key).push(c);
  }
  return map;
}
```

**关系复合**：`R∘S = { (a,c) | ∃b: (a,b)∈S ∧ (b,c)∈R }`。多跳权限「角色可继承」可用关系复合表达 — 传递闭包 `R*` 即「可达」。

## 与前端/算法的衔接

| 场景 | 数学结构 |
|------|----------|
| React key 列表去重 | 集合 |
| Redux selector 纯函数 | 映射 `state → view` |
| 路由 guard 权限矩阵 | 关系 `role × resource` |
| 依赖图拓扑排序 | 偏序 + DAG 遍历 |

哈希表在工程上实现映射 `key → value`：平均 O(1) 查找，依赖良好哈希函数与扩容策略；`Map` 键可为任意类型，比 plain object 更适合非字符串键。

## 有限集计数（面试口算）

| 问题 | 公式 |
|------|------|
| \|A\| | 元素个数 |
| \|A×B\| | \|A\|·\|B\| |
| \|P(A)\| | 2^\|A\| |

**容斥原理（两集）**：`|A∪B| = |A| + |B| − |A∩B|` — 统计「至少满足一条件」用户数时常用。

**三集容斥**：

```plaintext
|A∪B∪C| = |A|+|B|+|C| − |A∩B| − |A∩C| − |B∩C| + |A∩B∩C|
```

```javascript
// 同时满足 role=admin 与 region=cn → |A∩B|
// 至少满足其一 → 容斥，避免 double count
function unionSize(a, b) {
  return a.size + b.size - intersectionSize(a, b);
}
```

## 关系矩阵与权限（前端）

路由 guard 常写成 `can(role, action, resource)` — 本质是关系 `R ⊆ Roles × Actions × Resources`。

```typescript
type Perm = { role: string; resource: string; action: 'read' | 'write' };
const allowed = new Set(['admin:post:write', 'editor:post:write']);

function can(role: string, resource: string, action: string) {
  return allowed.has(`${role}:${resource}:${action}`);
}
```

**偏序**在任务依赖里：若 `A → B` 表示 B 依赖 A，则「先 A 后 B」的合法调度构成拓扑序 — 环存在则无合法全序。

**全序失败例**：两个 `z-index: auto` 的兄弟元素，层叠上下文规则给出比较，但并非全局全序 — 工程上常人为规定 stacking order。

## 双射与可逆 API 设计

| 映射性质 | 数学 | API 例子 |
|----------|------|----------|
| 单射 | 不同输入不同输出 | `id → user` 主键 |
| 满射 | 每个输出都有来源 | 枚举全覆盖 |
| 双射 | 一一对应可逆 | encode/decode 成对 |

```javascript
const byId = new Map(users.map((u) => [u.id, u]));
// 单射：id 唯一 → 反查 O(1)
```

权限矩阵 `role × resource` 是二元关系；feature flag 组合是笛卡尔积的子集 — 设计前先估 `|A×B|` 规模。

---

## 例题：笛卡尔积与测试组合

3 个浏览器 × 4 个分辨率 × 2 主题 = 24 种 UI 快照组合。全测不可行时用 **pairwise** 覆盖或优先级抽样：

```javascript
function cartesian(...arrays) {
  return arrays.reduce(
    (acc, cur) => acc.flatMap((a) => cur.map((b) => [...a, b])),
    [[]]
  );
}
// cartesian(['chrome','firefox'], ['1280','1920']).length === 4
```

集合运算 `A ∩ B` 对应「同时满足两 filter 的用户」；`|A ∪ B|` 用容斥避免重复计数 DAU。

## 小结

集合描述「有什么」，映射描述「怎么对应」，关系描述「怎么关联」。等价类与偏序是权限分组、任务调度、图算法的语言底座。

**易混点**：子集 `⊆` 允许相等，真子集 `⊂` 不允许；陪域（codomain）≠ 值域（range）；JS `Map` 是键值映射，不是数学上的集合；笛卡尔积有序、集合无序。

核对：`{1,2} × {a,b}` 有多少元素？若 `f(x)=x²` 从 `{−1,0,1}` 到 `{0,1}`，是否单射、是否满射？`|A∪B|` 如何用交表示？
