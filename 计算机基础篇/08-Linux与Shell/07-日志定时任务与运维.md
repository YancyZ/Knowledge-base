# 日志、定时任务与运维

生产问题常靠**日志**还原时间线；**cron/systemd timer** 跑备份与报表；**磁盘与日志轮转**避免把根分区写满 — 与 后端 08 · 部署排障、工程化 11 · 可观测性 形成「机器层 + 应用层」观测闭环。

## 日志体系

```mermaid
flowchart TB
  App[应用 stdout/stderr]
  Journal[journald]
  Files[/var/log/*]
  Ship[Agent Loki/ELK]
  App --> Journal
  App --> Files
  Files --> Ship
  Journal --> Ship
```

| 来源 | 查看 |
|------|------|
| systemd 服务 | `journalctl -u app -f ，since today` |
| 文件 | `tail -F /var/log/nginx/access.log` |
| Docker | `docker logs -f container` |

```bash
journalctl -u nginx --no-pager -n 200
grep '"status":500' /var/log/app/app.log | tail -20
```

结构化 JSON 日志便于 `jq` 过滤 — Node 用 pino/winston。

## logrotate

`/etc/logrotate.d/nginx` 示例逻辑：

```
/var/log/nginx/*.log {
  daily
  rotate 14
  compress
  delaycompress
  missingok
  notifempty
  sharedscripts
  postrotate
    systemctl reload nginx
  endscript
}
```

| 指令 | 作用 |
|------|------|
| `daily/weekly` | 轮转周期 |
| `rotate N` | 保留份数 |
| `compress` | gzip 旧日志 |
| `copytruncate` | 拷贝后截断（不支持 reload 的程序） |

**勿**只靠应用内按天切文件而不清理 — 磁盘满导致 DB 也不可写。

## cron 定时任务

```bash
crontab -e          # 当前用户
sudo crontab -u deploy -e
```

```
# 分 时 日 月 周 命令
0 2 * * * /opt/scripts/backup.sh >> /var/log/backup.log 2>&1
*/5 * * * * curl -sf http://localhost/health || mail -s alert admin@co
```

| 字段 | 范围 |
|------|------|
| 分 | 0-59 |
| 时 | 0-23 |
| 日 | 1-31 |
| 月 | 1-12 |
| 周 | 0-7（0=日） |

环境变量 minimal — cron 里用绝对路径，必要时脚本内 `source` profile。

## systemd timer（现代替代）

```ini
# /etc/systemd/system/backup.timer
[Unit]
Description=Daily backup

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

```bash
systemctl enable --now backup.timer
systemctl list-timers
```

比 cron 更易与 unit 集成、补跑 `Persistent` — 机器关机错过仍补执行。

## 磁盘与 inode 运维

```bash
df -h
df -i                 # inode 耗尽也无法写小文件
du -sh /var/* | sort -h | tail
find /var/log -type f -size +100M
```

| 告警 | 动作 |
|------|------|
| >85% 使用 | 清日志、扩盘、归档 |
| 大 core dump | `ulimit -c`、定期清理 |
| Docker 镜像 | `docker system prune`（谨慎） |

## 小结

journalctl 与文件日志是排障主线；logrotate 防磁盘爆；cron 与 systemd timer 执行周期任务 — 结构化日志 + 集中采集对接可观测平台。

**易混点**：cron 环境变量与交互 shell 不同；`copytruncate` 与 `reload` 选型；Docker 日志默认 json-file 也需驱动级轮转。

核对：如何只看今天某服务的 error？logrotate 与应用内按日切文件能否同时开？
