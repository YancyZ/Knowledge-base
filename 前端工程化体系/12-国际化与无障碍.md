# 12 · 国际化与无障碍

## 国际化（i18n）工程化

### 1.1 核心概念

| 术语 | 含义 |
|------|------|
| **i18n** | Internationalization — 国际化（使应用具备多语言能力） |
| **l10n** | Localization — 本地化（针对某地区翻译与适配） |
| **locale** | 语言+地区标识，如 `zh-CN`、`en-US`、`ar-SA` |
| **RTL** | Right-to-Left — 从右向左书写（阿拉伯语、希伯来语） |

### 1.2 不要硬编码文案

```tsx
// ❌
<button>提交</button>

// ✅
<button>{t('common.submit')}</button>
```

所有用户可见字符串进入语言包；含变量的用插值：

```json
{
  "welcome": "你好，{{name}}",
  "itemCount": "共 {{count}} 条"
}
```

```typescript
t('welcome', { name: 'Alice' });
t('itemCount', { count: 42 });
```

### 1.3 复数与性别

英语复数：

```json
{
  "apple": "{{count}} apple",
  "apple_plural": "{{count}} apples"
}
```

或使用 **ICU MessageFormat**：

```json
{
  "items": "{count, plural, =0 {无项目} one {# 个项目} other {# 个项目}}"
}
```

i18next、FormatJS 均支持 ICU。

## 技术方案选型

### 2.1 React 生态

| 库 | 特点 |
|----|------|
| **react-i18next** | 生态最大，Suspense 加载语言包 |
| **FormatJS (react-intl)** | ICU 标准，Yahoo 出品 |
| **Lingui** | 编译时提取，msg 宏 |

**react-i18next 示例**：

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';

i18n.use(initReactI18next).init({
  lng: 'zh-CN',
  fallbackLng: 'en-US',
  resources: {
    'zh-CN': { translation: zhCN },
    'en-US': { translation: enUS },
  },
  interpolation: { escapeValue: false }, // React 已转义
});
```

### 2.2 Vue 生态

**vue-i18n**：

```typescript
import { createI18n } from 'vue-i18n';

const i18n = createI18n({
  locale: 'zh-CN',
  fallbackLocale: 'en',
  messages: { 'zh-CN': zhCN, en: enUS },
});
```

Composition API：`const { t, locale } = useI18n()`。

### 2.3 语言包组织

```plaintext
locales/
├── zh-CN/
│   ├── common.json
│   ├── auth.json
│   └── dashboard.json
├── en-US/
│   └── ...
└── index.ts
```

按**业务模块**拆分，避免单文件万行。命名空间：`t('auth:login.title')`。

## 语言包工作流

### 3.1 提取与翻译

| 阶段 | 工具 / 做法 |
|------|-------------|
| 提取 | i18next-parser、Lingui extract、自定义 AST 扫描 |
| 翻译 | POEditor、Crowdin、Lokalise、Excel（小项目） |
| 审核 | 母语者 Review + 伪本地化测试 |
| 同步 | CI 检查未翻译 key |

### 3.2 伪本地化（Pseudo-localization）

将文案替换为加长、带变音符号的版本，检测：

- 布局被长文本撑破  
- 硬编码未进语言包的字符串  
- 截断问题  

```plaintext
Hello → [Ĥèļļō, world!]
```

### 3.3 动态加载

大应用按路由 lazy load 语言包：

```typescript
i18n.loadNamespaces('dashboard').then(() => i18n.changeLanguage('en-US'));
```

减少首屏 JSON 体积。

## locale 相关适配

### 4.1 日期与时间

禁止手写 `2024-01-01` 格式 — 用 **Intl** 或 dayjs/luxon locale：

```typescript
new Intl.DateTimeFormat('zh-CN', { dateStyle: 'long' }).format(date);
// 2024年1月1日
```

存储与传输用 **ISO 8601 UTC**；展示层按用户 locale 格式化。

### 4.2 数字与货币

```typescript
new Intl.NumberFormat('de-DE', { style: 'currency', currency: 'EUR' }).format(1234.5);
// 1.234,50 €
```

### 4.3 RTL 布局

```html
<html dir="rtl" lang="ar">
```

CSS 逻辑属性：

```css
.card {
  margin-inline-start: 16px; /* 替代 margin-left */
  padding-inline: 12px;
}
```

Tailwind：`ms-4`、`pe-2` 等 logical 工具类。

## 无障碍（a11y）基础

### 5.1 为什么重要

- **法律合规**：多国无障碍法规（如 WCAG、ADA、国标）  
- **用户覆盖**：视障、听障、运动障碍、认知障碍用户  
- **体验提升**：键盘导航、清晰对比度惠及所有人  

目标等级：**WCAG 2.1 AA** 为常见企业标准。

### 5.2 语义 HTML

```html
<!-- ✅ -->
<button type="button">保存</button>
<nav aria-label="主导航">...</nav>
<main>...</main>

<!-- ❌ -->
<div onclick="save()">保存</div>
```

标题层级 `h1`→`h2`→`h3` 不跳级；每页单一 `h1`。

### 5.3 表单

```html
<label for="email">邮箱</label>
<input id="email" type="email" aria-required="true" aria-invalid="true" aria-describedby="email-error" />
<span id="email-error" role="alert">格式不正确</span>
```

错误信息与字段 programmatically 关联。

### 5.4 键盘与焦点

- 所有交互可 Tab 到达  
- 可- 弹窗打开焦点 trap 在内，关闭还原  
- 快捷键不与系统/浏览器冲突  

### 5.5 图片与图标

```html
<img src="chart.png" alt="2024年Q1销售额增长15%的柱状图" />
<button aria-label="关闭"><IconClose /></button>
```

装饰性图片：`alt=""` 或 `role="presentation"`。

### 5.6 颜色对比度

正文文字与背景对比度 ≥ **4.5:1**（AA）；大字 ≥ 3:1。  
勿仅用颜色传达信息（红绿双色盲）— 配图标或文字。

工具：Chrome DevTools Accessibility、axe、Stark。

## WCAG 2.1 AA 工程对照

| 准则 | 要求 | 工程实现 |
|------|------|----------|
| 1.1 非文本内容 | 图片有替代文本 | `<img alt="">`；纯装饰图 `alt=""` |
| 1.3 可适应 | 信息不依赖单一感官 | 语义 HTML、heading 层级连续 |
| 1.4 可辨别 | 正文对比度 ≥ 4.5:1 | 设计 Token 阶段用对比度工具校验 |
| 2.1 键盘可操作 | 无鼠标可完成核心流程 | Tab 顺序、Esc 关闭浮层 |
| 2.4 易导航 | 跳过重复内容 | skip link、`main` landmark |
| 3.3 输入辅助 | 错误可理解、可纠正 | `aria-describedby` + `role="alert"` |
| 4.1 兼容 | 可被辅助技术解析 | 合法 HTML、ARIA 不滥用 |

**常见违规**：`<div onClick>` 无键盘支持；placeholder 代替 label；`<h1>` 跳 `<h3>`。

## 键盘与焦点

### 9.1 Tab 顺序原则

- 优先原生可聚焦元素：`<button>`、`<a href>`、`<input>`
- `tabindex="0"` 仅当必须让非交互元素可聚焦；**禁止** `tabindex > 0` 人为跳序
- 模态打开：焦点移入首可聚焦元素；关闭：焦点还原到触发按钮

### 9.2 Focus Trap（模态内）

Tab 在模态首尾循环，防止焦点「漏」到背后页面。Headless UI（Radix、React Aria）已内置，自研须手写 `keydown` 监听。

### 9.3 常用 ARIA 组合

| UI | 属性 |
|----|------|
| 模态 | `role="dialog"` `aria-modal="true"` `aria-labelledby` |
| 下拉 | `aria-expanded` `aria-controls` |
| 加载 | `aria-busy="true"` |
| 动态提示 | `aria-live="polite"` 或 `assertive`（仅紧急错误） |

## i18n + a11y 交叉点

- 读屏软件朗读语言随 `lang` 属性 — 切换 locale 时更新 `<html lang>`  
- 翻译后文本变长 — 布局弹性  
- RTL 下 icon 方向镜像（箭头、进度）  

---

## i18n 端到端流程

1. **开发**：用户可见文案一律 `t('key')`  
2. **CI**：i18n-parser 检查缺失翻译、僵尸 key  
3. **翻译**：导出 JSON/xliff → TMS → 回写各 locale  
4. **QA**：伪本地化 + 各语言冒烟 + RTL 布局检查  
5. **发布**：语言包版本与 app `release` 对齐，便于回滚

## 伪本地化（Pseudo-localization）

开发环境启用伪翻译（拉长 30% 字符串、加 accent）— 提前暴露布局截断与未翻译 key：

```json

```

CI 可对比 `en-US` 与 `zh-CN` key 集合差集。

## 小结

i18n 与 a11y 都是**产品可达性**：语言切换不硬编码文案；键盘、读屏、对比度、焦点管理是合规与体验的共同要求。

i18n key + 懒加载语言包；RTL 用逻辑属性；aria 补充而非替代原生；focus trap 于模态；eslint-plugin-jsx-a11y。

**易混点**：拼接翻译字符串破坏语序；aria-hidden 藏可聚焦元素；仅颜色传达状态。

核对：Tab 能否走通主流程？lang 属性是否与内容一致？
